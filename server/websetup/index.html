Setup Is Not Available In This Build

The included clients do not have production crypto yet, so they cannot create the first owner safely.

Keep this instance private. Deploy a release with production crypto, then create the owner from that client. Never use a placeholder or test key package.

The production client will call POST /api/v1/setup/owner with a real device_key_package. Remote setup also requires the one-time token from PRIVATE_MESSENGER_SETUP_TOKEN.
